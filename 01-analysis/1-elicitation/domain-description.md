# Описание домена и ограничения

> Этап 1. Артефакт 1 проекта: согласованное описание домена + список ограничений.
> Заполняется после ответов заказчика ([customer-questions.md](customer-questions.md)).

## Описание домена

**Бизнес.** «Глина» — небольшая гончарная мастерская в центре города, работающая
третий год. Мастерская организует **групповые мастер-классы** по лепке и работе
на гончарном круге. Занятие длится 2–2.5 часа и проводится по одной из **программ**:
лепка для новичков или работа на гончарном круге для тех, кто уже пробовал. Каждое
занятие ведёт **мастер** (сейчас их четверо, в сезон нанимаются помощники).

**Процесс.** Владелица заранее формирует **расписание на неделю вперёд**: для каждого
занятия задаются дата и время начала, программа, назначенный мастер и число мест. Мастерская
располагает **10 гончарными кругами**; для новичковых групп берут не более 6 человек
(нужно больше внимания каждому), для работы на круге — до 10 (по числу кругов). Клиент
видит свободные **слоты** занятий и **записывается**, указывая, придёт ли со своими
инструментами и фартуком или возьмёт **напрокат**. До занятия клиент может **отменить**
запись — при ранней отмене место освобождается, поздняя отмена оставляет заготовленную
глину и круг простаивать. Сейчас запись клиентов ведётся вручную (Instagram-директ +
ежедневник), что приводит к двойным записям и путанице, особенно в выходные.

**Цель клиентского приложения.** Дать клиентам самостоятельную онлайн-запись на
мастер-классы (просмотр и фильтрация слотов, бронирование, выбор инструментов, отмена,
свои бронирования), исключив ручную путаницу и двойные брони. Управление расписанием
и работу мастеров/владелицы обеспечивает **существующая инфраструктура**; приложение
потребляет её данные через API.

**Ключевые объекты домена:**

- **Занятие / слот** — конкретный мастер-класс: дата/время начала, программа, мастер,
  всего и свободно мест. Создаётся в существующей инфраструктуре; клиенту доступен только
  для чтения.
- **Программа** — вариант занятия (лепка / гончарный круг), определяет длительность
  и потолок мест (справочные данные, read-only).
- **Мастер** — ведёт занятие; для клиента — справочные данные слота (read-only).
- **Инструменты** — гончарный набор (инструменты, фартук); всего 10 комплектов; может
  выдаваться напрокат.
- **Запись (бронь)** — резерв места клиентом на конкретный слот: число мест,
  свои/прокатные инструменты, статус.
- **Клиент** — записывается на занятия, отменяет запись.

## Пользователи и роли

> **Скоуп текущей работы — только клиентское мобильное приложение и API для него.**
> Роли Мастер и Владелица/Администратор обслуживаются **существующей инфраструктурой**
> (бэкенд, расписание, админка) и в этот скоуп не входят.

| Роль | Описание | Цели | В скоупе |
| :-- | :-- | :-- | :-- |
| **Клиент** | Гость мастерской, учится лепить и работать на круге. Пользуется с телефона. | Увидеть и отфильтровать свободные слоты, записаться (себя и гостей), выбрать свои/прокатные инструменты, видеть свои бронирования, отменить запись, получать напоминания. | **Да** |

> Мастер и Владелица/Администратор взаимодействуют с системой через **уже реализованные**
> интерфейсы существующей инфраструктуры. Клиентское приложение лишь потребляет подготовленные
> ими данные (слоты, программы, мастера) через API.

## Ограничения

**Бизнес-ограничения**

- Всего **10 гончарных кругов** — общий потолок инвентаря и прокатного фонда.
- Новичковая группа (лепка) — **не более 6 человек**; группа по кругу — до 10
  (по числу кругов).
- Лимит мест слота = `min(свободные круги, потолок программы)`.
- **Политика отмен:** бесплатная отмена не позднее чем за **2 часа** до начала;
  поздняя отмена (<2 ч) фиксируется без освобождения места и без денежных штрафов.
- **Оплата офлайн** (наличные / перевод на карту) — приложение лишь фиксирует запись и
  показывает цену.
- **Период списка слотов по умолчанию — 7 дней (R-027).** По умолчанию клиенту показываются
  слоты на **ближайшие 7 дней** от текущего момента — это совпадает с горизонтом планирования
  расписания («на неделю вперёд»). Более длинный период доступен через **фильтр дат**. При
  отсутствии расписания на ближайшие дни показывается **empty state**
  («Пока нет доступных занятий»).
- **Отмена занятия мастерской / по форс-мажору (R-008).** Если занятие отменяет мастерская
  (организатор или из-за поломки оборудования), бронь на стороне клиента **не удаляется**,
  а получает статус **«Отменено мастерской»** с указанием причины; клиенту приходит
  **push-уведомление**. **Повторная запись на отменённый слот запрещена.** Само действие
  отмены слота выполняется в существующей инфраструктуре (вне скоупа клиентского приложения).

**Технические / платформенные**

- **Mobile-first**: клиентское приложение рассчитано на телефон (использование в мастерской).
- Клиентское приложение взаимодействует с **существующим бэкендом через API**; слоты, программы
  и мастера доступны только для чтения.
- **Граница серверной интеграции (R-004).** Детали серверной реализации — транзакционность,
  SLA, маппинг сущностей на внутренние модели бэкенда — **вне скоупа клиентского приложения**.
  Существующий бэкенд — **black-box источник истины**: гарантия «**0 двойных броней**»
  обеспечивается на его стороне (атомарная проверка свободных мест при бронировании). Со стороны
  клиента граница закрывается **контрактом API**: приложение полагается на ответы бэкенда и
  корректно обрабатывает отказ, если мест уже нет.
- **Данные существующей инфраструктуры (R-015).** Это **учебный/тестовый проект**,
  исторических данных и легаси-схемы нет. **Канонической схемой считается контракт API**
  (OpenAPI в `01-analysis/api/`): бэкенд по условию отдаёт ровно те поля, что описаны в
  контракте, включая `rental_price`, `program_description`, `workshop_address` и
  `free_rental_kits`. Поэтому **миграция/backfill и стратегия дозаполнения исторических
  слотов вне скоупа** — отдельных требований к ним не формулируем.

**Проект**

- **Срок** — к началу осеннего сезона, ~2 месяца (жёсткая цель).
- **Бюджет** ограничен → начинаем с самого нужного, «навороты» позже.

**Границы скоупа**

Скоуп — **клиентское мобильное приложение и API для него**. Управление расписанием, роли
мастера и владелицы, оценки, админка — это **существующая инфраструктура**, вне скоупа.

| В скоупе клиентского приложения | Вне скоупа (существующая инфраструктура / не делаем сейчас) |
| :-- | :-- |
| Регистрация/вход клиента по имени и телефону | Роли Мастера и Владелицы/Администратора, админ-экран |
| Просмотр и **фильтрация** списка слотов (дата, тип программы, наличие мест, мастер) | Создание и редактирование расписания (слоты приходят из существующего бэкенда через API) |
| Самостоятельная запись, в т.ч. несколько мест за раз | Оценка мастера и рейтинги — **Phase 2** (R-001; заказчица ранее называла функцию критичной, перенос в Phase 2 согласован с заказчицей 2026-07-01) |
| Выбор «свои / прокатные инструменты», контроль лимита мест и прокатного фонда | Отметка явки/неявки, отмена слота мастером + массовое уведомление |
| Список своих бронирований | Онлайн-оплата; ручная отметка «оплачено» |
| Отмена записи с правилом 2 часов | Автоматическая проверка форс-мажора |
| Цена и фиксация записи (оплата офлайн) | Программа лояльности для постоянных клиентов |
| Напоминания о записи — по возможности в срок | Публичный показ рейтинга клиентам — **Phase 2** (R-001) |

## Глоссарий

| Термин | Определение |
| :-- | :-- |
| **Занятие / слот** | Запланированный групповой мастер-класс: дата/время начала, программа, мастер, всего и свободно мест. |
| **Программа** | Вариант занятия: лепка для новичков (потолок 6) или работа на гончарном круге (до 10). |
| **Группа** | Состав клиентов, записанных на один слот. |
| **Инструменты** | Гончарный набор из инвентаря мастерской; всего 10 комплектов; может выдаваться напрокат. |
| **Аренда / прокат** | Выдача клубного набора инструментов клиенту, у которого нет своих. |
| **Свои инструменты** | Клиент приходит с собственными инструментами и фартуком; занимает место в группе, но не прокатный фонд. |
| **Запись (бронь)** | Резерв одного или нескольких мест на конкретный слот: число мест, свои/прокатные инструменты, статус. |
| **Отмена** | Снятие записи. Клиентом: ранняя (≥2 ч до начала) освобождает место; поздняя (<2 ч) фиксируется без освобождения места. Мастерской (организатором / по форс-мажору): бронь переходит в статус «Отменено мастерской» с причиной, клиент получает push, повторная запись на этот слот запрещена. |
| **Отмена мастерской** | Отмена занятия организатором или из-за форс-мажора (выполняется в существующей инфраструктуре). На стороне клиента бронь сохраняется со статусом «Отменено мастерской» + причина; отправляется push-уведомление; повторная запись на отменённый слот недоступна. |
| **Администратор / владелица** | Марина — управляет расписанием и мастерами через существующую инфраструктуру (вне скоупа клиентского приложения). |
| **MVP** | Минимально жизнеспособный продукт: набор функций к старту сезона. |
