# Нефункциональные требования

> Этап 2. Как система должна работать (качества, ограничения, стандарты).
> Не путать с функциональными ([functional-requirements.md](functional-requirements.md)).

## Производительность

| ID | Требование | Источник |
| :-- | :-- | :-- |
| NFR-1 | Время ответа API для списка слотов — ≤500 мс (p95) при нагрузке до 100 RPS | Ограничение платформы |
| NFR-2 | Время ответа API для создания брони — ≤800 мс (p95); атомарная операция на стороне бэкенда | R-004 |
| NFR-3 | Время запуска приложения (cold start) — ≤3 сек на устройстве среднего сегмента | Mobile-first |

## Надёжность

| ID | Требование | Источник |
| :-- | :-- | :-- |
| NFR-4 | Доступность API — ≥99.5% (плановые окна обслуживания вне пиковых часов) | Бизнес-ограничения |
| NFR-5 | При недоступности сети приложение показывает кэшированные слоты (последняя успешная загрузка) с индикатором «обновление недоступно» | Mobile-first; UX |

## Безопасность

| ID | Требование | Источник |
| :-- | :-- | :-- |
| NFR-6 | Передача данных по TLS 1.2+; API — HTTPS only | Стандарт |
| NFR-7 | Аутентификация по Bearer JWT; срок жизни access-токена — 24 ч, refresh — 30 дней | FR-2 |
| NFR-8 | Хранение push-токенов в защищённом хранилище устройства (Keychain / Keystore) | FR-33 |
| NFR-9 | Персональные данные (имя, телефон) — в соответствии с 152-ФЗ; хранение и обработка на стороне бэкенда; клиент не кэширует чужие ПДн | R-015; законодательство |
| NFR-10 | СМС-otp: длина кода — 6 цифр; TTL — 5 мин; лимит повторной отправки — 1 раз в 60 сек; макс. попыток ввода — 3; нормализация телефона — E.164 (+7...) | FR-43 |
| NFR-11 | Защита от перебора OTP: блокировка номера на 30 мин после 5 неудачных попыток в течение часа | FR-43 |
