# User Stories

> Этап 2. Истории с приёмочными критериями (AC) — «как <роль>, я хочу <цель>, чтобы <ценность>».

## US-1. Регистрация

**Как** потенциальный клиент,  
**я хочу** зарегистрироваться по имени и телефону,  
**чтобы** получить доступ к записи на занятия.

**Приёмочные критерии:**
- [ ] Поля: имя (1–100 символов), телефон (российский формат, +7...)
- [ ] После ввода телефона система отправляет SMS с 6-значным кодом
- [ ] Код действителен 5 мин
- [ ] При верном коде создаётся аккаунт и выдаётся JWT
- [ ] При повторном входе по тому же телефону — авторизация без создания дубля

## US-2. Просмотр слотов

**Как** клиент,  
**я хочу** видеть список предстоящих занятий на неделю вперёд,  
**чтобы** выбрать подходящее.

**Приёмочные критерии:**
- [ ] Список сортирован по времени начала (ближайшие выше)
- [ ] Каждый элемент: дата/время, программа, мастер, свободно/всего мест, цена
- [ ] Заполненные и отменённые слоты показываются с пометкой «Мест нет», CTA неактивна
- [ ] При отсутствии слотов — сообщение «Пока нет доступных занятий» с иконкой/иллюстрацией

## US-3. Фильтрация слотов

**Как** клиент,  
**я хочу** фильтровать занятия по дате, программе и мастеру,  
**чтобы** быстрее найти подходящее.

**Приёмочные критерии:**
- [ ] Пресеты дат: «Сегодня», «Эта неделя», «Выходные», кастомный диапазон
- [ ] Фильтр программы: чекбоксы «Лепка», «Гончарный круг» (внутри — ИЛИ)
- [ ] Фильтр мастера: выпадающий список или поиск; если справочник пуст — скрыт
- [ ] Опция «Только свободные» — тумблер, по умолчанию выключен
- [ ] Активные фильтры отображаются на экране; есть кнопка «Сбросить»

## US-4. Запись на занятие

**Как** клиент,  
**я хочу** записаться на выбранное занятие с выбором инструментов,  
**чтобы** забронировать место.

**Приёмочные критерии:**
- [ ] На экране записи: выбор числа мест (1–3, ограничено `max_seats`)
- [ ] Для каждого места: выбор «Свои инструменты» / «Прокат» (радио-кнопка или сегмент)
- [ ] Если прокатных наборов недостаточно — опция проката недоступна с пояснением
- [ ] Итоговая цена отображается из `price_total` API
- [ ] При успешной записи — экран подтверждения с деталями брони
- [ ] При неудаче (мест нет, 410) — понятное сообщение с предложением выбрать другой слот

## US-5. Отмена записи

**Как** клиент,  
**я хочу** отменить свою запись,  
**чтобы** освободить место, если планы изменились.

**Приёмочные критерии:**
- [ ] Отмена доступна до начала занятия
- [ ] При ранней отмене (≥2 ч) — бронь переходит в «Отменена», места освобождаются
- [ ] При поздней отмене (<2 ч) — бронь переходит в «Поздняя отмена», места не освобождаются
- [ ] Отмена только целиком; частичная отмена места/гостя не поддерживается
- [ ] Подтверждение отмены через диалог «Вы уверены?»

## US-6. Мои бронирования

**Как** клиент,  
**я хочу** видеть историю своих записей,  
**чтобы** не забыть о предстоящих занятиях.

**Приёмочные критерии:**
- [ ] Список отсортирован по времени начала; разделение «Предстоящие» / «Прошедшие»
- [ ] Каждая бронь: дата/время, программа, мастер, статус, число мест, тип инструментов
- [ ] Статусы: «Активна», «Отменена», «Поздняя отмена», «Отменено мастерской» (+ причина)
- [ ] Пагинация при большом количестве записей

## US-7. Push-уведомления

**Как** клиент,  
**я хочу** получать напоминания о занятиях,  
**чтобы** не забыть прийти.

**Приёмочные критерии:**
- [ ] При первом входе — запрос разрешения на push
- [ ] Напоминания: за 24 ч и за 2 ч до начала
- [ ] При отмене мастерской — push с причиной отмены
- [ ] Тап на push открывает приложение на соответствующем экране
- [ ] При отказе от push приложение работает штатно; напоминания не доставляются

## US-8. Профиль

**Как** клиент,  
**я хочу** просматривать и редактировать свой профиль,  
**чтобы** мои данные были актуальны.

**Приёмочные критерии:**
- [ ] Поля: имя, телефон (только просмотр)
- [ ] Редактирование имени
- [ ] Выход из аккаунта (logout)
- [ ] Удаление аккаунта с подтверждением; активные брони аннулируются, прошедшие анонимизируются
