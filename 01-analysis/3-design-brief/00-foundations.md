# Требования на дизайн · Foundations (сквозные правила)

> **Этап 3.** Сквозной документ дизайн-требований приложения «Глина». Описывает принципы,
> структурные токены, паттерны навигации и состояний, доступность и микрокопию, **общие для
> всех экранов**. Экранные документы ссылаются сюда и не дублируют эти правила.

**Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-07-01 · **Зона:** НЗ + АЗ

## 1. Продукт и аудитория

**«Глина»** — клиентское мобильное приложение для самостоятельной записи на мастер-классы
гончарной мастерской. Заменяет запись через Instagram-директ и бумажный ежедневник.

**Единственная роль — «Клиент».** Мастер и владелица в приложение не входят. Справочные
данные (слоты, программы, мастера) — read-only из API. Оплата — **офлайн** (наличные /
перевод); приложение показывает цену и фиксирует запись, онлайн-оплаты нет.

**Контекст использования:** клиент приходит в мастерскую или планирует визит с телефона;
руки могут быть в глине, освещение варьируется. Это диктует крупные элементы, высокий
контраст и минимум шагов (NFR-1).

## 2. Дизайн-принципы

| # | Принцип | Источник | Что это значит для макета |
|---|---|----------|---------------------------|
| P1 | **Mobile-first для мастерской** | NFR-1 | Крупные тач-зоны, высокий контраст, читаемость в разном освещении, минимум мелкого текста. |
| P2 | **Короткий путь к записи** | NFR-2 | От списка до подтверждения — **≤ 3 экранов**. Не добавлять необязательных шагов/полей. |
| P3 | **Минимальный порог входа** | NFR-3 | Регистрация — только имя + телефон, без пароля. Не запрашивать лишних данных. |
| P4 | **Воспринимаемая скорость** | NFR-6 | Скелетоны вместо пустого экрана; отклик списка и подтверждения ощущается < 2–3 с. |
| P5 | **Только свои данные** | NFR-9, NFR-11 | Клиент видит лишь свои записи и контакты; в UI нет доступа к чужим/админским данным. |
| P6 | **Честность и спокойствие** | UC-1/UC-2 | Ошибки и правила (места, инструменты, 2 часа) объясняются понятно и без давления; штрафов нет. |

## 3. Структурные токены (без бренда)

### 3.1 Тач-зоны и размеры
- Минимальный размер интерактивного элемента — **≥ 44–48 pt** по меньшей стороне.
- Основной CTA — во всю ширину контентной области, высота не менее минимальной тач-зоны.
- Между кликабельными элементами — отступ, исключающий промахи.

### 3.2 Контраст и читаемость (NFR-1)
- Контраст текста к фону — не ниже **WCAG AA** (обычный текст ≥ 4.5:1, крупный ≥ 3:1).
- Состояния не передаются **только цветом** — дублируются иконкой/текстом/формой.
- Важные числа (свободно мест, цена, время начала) — крупные и контрастные.

### 3.3 Типографическая иерархия
- **Заголовок экрана** → **Заголовок секции/карточки** → **Основной текст** → **Вторичный/подпись**.
- Минимальный размер основного текста комфортен для чтения.

### 3.4 Иконки и индикаторы
- Иконки сопровождаются текстом в ключевых местах (таб-бар, статусы).
- Индикатор активных фильтров, бейдж статуса записи — визуально считываемы с первого взгляда.

## 4. Каркас экрана и навигация

### 4.1 Базовый каркас
```
┌─────────────────────────────┐
│ Хедер (заголовок / назад)   │  ← фиксированный
├─────────────────────────────┤
│                             │
│ Скролл-контент              │  ← основная зона
│                             │
├─────────────────────────────┤
│ Фикс. нижний CTA (если есть)│  ← всегда виден
└─────────────────────────────┘
│ Таб-бар (в АЗ, верхнеуровн.)│  ← только на корневых экранах
└─────────────────────────────┘
```

### 4.2 Таб-бар (авторизованная зона)
Три верхнеуровневых раздела:
- **Занятия** (SCR-002) — стартовая вкладка.
- **Мои записи** (SCR-005).
- **Профиль** (SCR-007).

Таб-бар скрывается на вложенных экранах. Каждая иконка сопровождается подписью.

### 4.3 Bottom Sheet (шторки BS-001 / BS-002 / BS-003)
Единые правила для всех шторок:
- Высота — по контенту, но не выше ~90% экрана; длинный контент скроллится внутри.
- **Бэкдроп** (затемнение фона) + закрытие по тапу вне шторки.
- **Swipe-to-close** жестом вниз + видимый «грабер» сверху.
- Явная кнопка закрытия/отмены.

### 4.4 Карта адреса мастерской (статичный превью Яндекс.Карт)
Сквозной компонент для экранов с адресом (SCR-003, SCR-006).

```
┌─────────────────────────────────┐
│  ░░░░░ карта (статичный превью) ░░│
│  ░░░░░ 📍 ░░░░░░ адрес мастерской ░│  ← пин на карте
│  [ Открыть карту › ]            │  ← тап → интерактивная карта
├─────────────────────────────────┤
│  📍 Адрес мастерской            │
│  ул. Гончарная, 12              │
│  м. Третьяковская               │
└─────────────────────────────────┘
```

- **Тип:** статичное изображение карты (Яндекс Static API) с пином адреса.
- **Тап по карте** → полноэкранная интерактивная карта.
- **Текстовый блок адреса** под картой обязателен — fallback при недоступности карты.

## 5. Сквозной паттерн состояний экрана

| Состояние | Что показываем | Правило |
|-----------|----------------|---------|
| **Loading** | Скелетон/шиммер в форме будущего контента | Не пустой белый экран. |
| **Content** | Данные | Основной сценарий. |
| **Empty** | Заглушка + понятная подсказка | Объясняет, почему пусто, и что сделать. |
| **Error** | Заглушка ошибки + кнопка «Обновить» | Нейтральный тон; даёт повтор. |

## 6. Tone of voice и общая микрокопия

**Тон:** простой, прямой, дружелюбный. Обращение на «вы». Без вины и давления.

**Сквозные тексты:**

| Контекст | Текст |
|----------|-------|
| Оплата | «Оплата на месте: наличные или перевод на карту.» |
| Лейблы инструментов | «Свои инструменты» / «Прокатный набор» |
| Правило отмены | «Отмена не позднее чем за 2 часа до начала — место освобождается. Позже — место остаётся за вами, но штрафов нет.» |
| Поздняя отмена (итог) | «Поздняя отмена: место не освобождено (правило 2 часов). Штраф не взимается.» |
| Кнопка повтора | «Обновить» |
| Сетевая ошибка (общая) | «Не удалось загрузить. Проверьте соединение и попробуйте снова.» |

## 7. Доступность (WCAG 2.1 AA)

- **Контраст:** WCAG AA.
- **Тач-зоны:** ≥ 44 pt.
- **Dynamic type:** поддержка системного увеличения шрифта.
- **Screen reader:** все интерактивные элементы имеют accessibleLabel.
- **Не только цвет:** состояния дублируются иконкой/текстом.

## 8. Сквозные функции

### 8.1 Push-уведомления (FR-33)
- Напоминания за 24 ч и за 2 ч до начала.
- Уведомление об отмене мастерской.
- Запрос разрешения — после первой успешной записи (на шторке BS-002).
- Тап на push → переход в приложение на соответствующий экран.

### 8.2 Офлайн и сетевые ошибки
- **Просмотр кэша офлайн разрешён** — с пометкой устаревания.
- **Мутации офлайн запрещены** — блокируются с сообщением «нет сети».
- **Таймаут запроса — ~10 с.**

## 9. Глоссарий

| Термин | Значение |
|--------|----------|
| **Занятие / Слот** | Конкретный мастер-класс: дата, время, программа, мастер, цена, места. |
| **Программа** | Вариант занятия (лепка / гончарный круг); свой потолок мест. |
| **Инструменты** | Гончарный набор: прокатный (из фонда) или собственный (клиент со своими). |
| **Запись (бронь)** | Бронь на слот: число мест, вариант инструментов, статус. |
| **Ранняя отмена** | ≥ 2 ч до начала → места возвращаются. |
| **Поздняя отмена** | < 2 ч → запись фиксируется, место не освобождается. |
