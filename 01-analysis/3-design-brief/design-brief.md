# Бриф для UI/UX дизайнера · «Глина»

> Требования на дизайн клиентского мобильного приложения «Глина» (самостоятельная запись на
> мастер-классы гончарной мастерской). Этот файл — **обзор и индекс**: единые правила
> вынесены в [00-foundations.md](00-foundations.md), детальные требования — в отдельный
> документ на каждый экран.

**Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-07-01

## Цель и контекст

**«Глина»** заменяет ручную запись на мастер-классы через Instagram-директ и бумажный
ежедневник, устраняя двойные брони и путаницу. Клиент самостоятельно просматривает слоты,
фильтрует, записывается (себя и гостей), выбирает вариант инструментов, отменяет записи
и получает напоминания. Платформа — **нативное/гибридное мобильное приложение iOS/Android**
(mobile-first), распространяется через App Store и Google Play.

**Скоуп — только роль «Клиент».** Мастер и владелица работают через существующую
инфраструктуру/админку и в приложение не входят.

**Что входит в MVP:**
- Профиль клиента: просмотр/редактирование имени и телефона, выход, удаление аккаунта.
- Цена и прокат: прокат — отдельный тариф; итог = `цена × места + прокат × наборы`. Свои инструменты бесплатны.
- Адрес мастерской на карте с fallback-текстом.
- Отмена — только целиком.

## Роли и их задачи

| Роль | Что делает в приложении |
| :-- | :-- |
| **Клиент** | Просматривает/фильтрует слоты, записывается, выбирает инструменты, смотрит цену, отменяет, видит брони, получает напоминания. |
| Мастер / Владелица | **Не входят в приложение** — работают через существующую инфраструктуру. |

## Пользовательские сценарии (user flows)

- **Вход:** запуск → SCR-001 (телефон → код из SMS → имя) → SCR-002.
- **Запись:** SCR-002 → SCR-003 → SCR-004 → BS-002 → (SCR-005). ≤ 3 экранов до подтверждения. После первой записи в BS-002 запрашивается push.
- **Фильтрация:** SCR-002 → BS-001 → SCR-002 (с применёнными фильтрами).
- **Отмена:** SCR-005 → SCR-006 → BS-003 → SCR-006 (с обновлённым статусом).
- **Профиль:** SCR-007 — редактирование, выход, удаление.

## Состояния экранов

Единый паттерн: Loading (скелетон) → Content → Empty → Error (+ «Обновить»). Описан в
[00-foundations.md §5](00-foundations.md).
