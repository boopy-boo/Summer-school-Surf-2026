## Обзор

SCR-001 — точка входа для неавторизованного пользователя. Обслуживает регистрацию и вход по одному потоку через SMS OTP:

1. **Шаг 1 — Телефон:** ввод номера → `sendOtp` (POST `/auth/otp/send`).
2. **Шаг 2 — Код из SMS:** ввод кода → `verifyOtp` (POST `/auth/otp/verify`) → ответ содержит `tokens`, `user`, `is_new`.
3. **Шаг 3 — Имя (только новый пользователь, `is_new = true`):** ввод имени → `updateProfile` (PATCH `/profile`).

Таб-бар отсутствует. При наличии пары токенов экран пропускается.

### Входящая

| Источник | Триггер | Условие |
|----------|---------|---------|
| Запуск приложения | Открытие | Нет пары токенов в защищённом хранилище |
| SCR-007 Профиль | «Выйти» / «Удалить аккаунт» | Сессия завершена |

### Исходящая

| Назначение | Триггер |
|------------|---------|
| SCR-002 Список слотов | Успешный вход (`verifyOtp` 200, `is_new=false`) |
| SCR-002 Список слотов | Завершение регистрации (`updateProfile` 200) |

## Входные данные

| Название | Тип | Описание |
|----------|-----|----------|
| `tokens` | Keychain / Keystore | Пара токенов (`access_token` + `refresh_token`) |
| `phone` | Состояние флоу | Введённый номер, E.164 |
| `is_new` | Состояние флоу | Флаг нового пользователя из `verifyOtp` |

## Инициализация

На старте — локальная проверка пары токенов. Сетевых запросов нет.

```
onEnter → Проверка токенов
├── Токены есть → SCR-002 (пропуск)
└── Токенов нет → Шаг 1 (ввод телефона)
```

## Используемые запросы

### sendOtp

**POST `/auth/otp/send`**  
**Триггер:** Тап «Получить код»

| Параметр | Тип | Источник |
|----------|-----|----------|
| `phone` | string | Поле «Телефон» |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Успех | 204 | Переход на шаг 2 |
| 429 | too_many_requests | «Попробуйте через 60 секунд» |
| Сеть | Нет соединения | Снек из foundations |

### verifyOtp

**POST `/auth/otp/verify`**  
**Триггер:** Тап «Подтвердить»

| Параметр | Тип | Источник |
|----------|-----|----------|
| `phone` | string | Состояние `phone` |
| `code` | string | Поле «Код из SMS» |
| `name` | string | Поле «Имя» (для нового) |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Успех (известный) | 200, `is_new=false` | Сохранить токены → SCR-002 |
| Успех (новый) | 200, `is_new=true` | Сохранить токены → Шаг 3 |
| 401 | Неверный/истёкший код | «Код неверен или просрочен» |
| 429 | Лимит попыток | «Слишком много попыток» |
| Сеть | Нет соединения | Снек из foundations |

### updateProfile

**PATCH `/profile`** (шаг 3, только `is_new=true`)  
**Триггер:** Тап «Продолжить»

| Параметр | Тип | Источник |
|----------|-----|----------|
| `name` | string | Поле «Имя» |

**Заголовок:** `Authorization: Bearer <access_token>`

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Успех | 200 | Переход на SCR-002 |
| 400 | Невалидное имя | Подсветка поля |
| 401 | Токен просрочен | Refresh → повтор; если не удался → шаг 1 |

| Состояние | Условие | Отображение |

## Критерии приёмки

### Позитивные

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-001 | При отсутствии токенов показывается шаг 1 (только телефон, без пароля) | P0 |
| AC-002 | При наличии токенов SCR-001 пропускается, сразу SCR-002 | P0 |
| AC-003 | Валидный телефон → код приходит, открывается шаг 2 | P0 |
| AC-004 | Известный номер (`is_new=false`) → после кода сразу SCR-002 | P0 |
| AC-005 | Новый номер (`is_new=true`) → после кода шаг 3 → имя → SCR-002 | P0 |

### Негативные

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-N01 | Невалидный телефон → CTA disabled, запрос не отправляется | P0 |
| AC-N02 | Неверный код → «Код неверен или просрочен» | P0 |
| AC-N03 | Пустое имя на шаге 3 → «Укажите имя», запрос не отправляется | P1 |
| AC-N04 | Нет сети → снек, данные сохранены | P1 |
