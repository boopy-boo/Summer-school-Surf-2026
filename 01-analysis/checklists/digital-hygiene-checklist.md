# Чеклист цифровой гигиены

## Архитектура документации

- [ ] Каждый документ имеет **заголовок и описание** — что это и зачем
- [ ] Нумерация требований **сквозная** — ID не переиспользуются при удалении/переносе
- [ ] **Ссылки между документами** работают: ответы на вопросы → требования → use cases → API
- [ ] **Трассировка** от требования до API-эндпоинта присутствует

## Качество текста

- [ ] **Без орфографических ошибок** — проверено
- [ ] **Без противоречий** внутри и между документами
- [ ] **Терминология единая** — глоссарий соблюдён
- [ ] **Формулировки конкретные** — без «возможно», «как-нибудь», «потом разберёмся»
- [ ] **Числовые ограничения** заданы явно (лимиты, тайминги, коды ответов)

## API

- [ ] OpenAPI 3.0.3 — валидный синтаксис
- [ ] Каждый эндпоинт имеет `operationId`
- [ ] Все ответы описаны, включая ошибки (4xx, 5xx)
- [ ] Примеры значений присутствуют
- [ ] Ссылки `$ref` рабочие (относительные пути)

## Скоуп и границы

- [ ] **Вне скоупа** явно перечислено
- [ ] **Phase 2** — функции перенесённые, а не потерянные
- [ ] **Решения** зафиксированы с датой и контекстом

## Рецензирование

- [ ] Прочитан «свежим взглядом» через день после написания
- [ ] Проверена трассировка: от брифа → вопросов → требований → API
