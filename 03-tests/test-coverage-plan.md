# 🧪 Высокоуровневый план покрытия тестирования — «Глина» (iOS + Android)

**Версия:** 1.0 · **Дата:** 2026-07-05  
**Источник:** Артефакты `01-analysis/`, выявленные риски из `tz-review-report.md`  
**Цель:** Определить полный набор тестовых сценариев, необходимых для валидации клиентского приложения и API перед релизом MVP.

---

## 📐 Методология покрытия

| Уровень | Тип тестов | Инструменты (пример) | Объём |
|---------|-----------|----------------------|-------|
| L1 | Unit / Component | XCTest, JUnit, Mockito | Ядро: валидаторы, мапперы, маппинг API→Domain |
| L2 | API Contract / Integration | Postman/Newman, REST Assured, Swagger Validator | Все эндпоинты + коды ошибок |
| L3 | UI / E2E (автоматизация) | Appium, Maestro, XCUITest, Espresso | Критический путь (SCR + BS) |
| L4 | NFR / Security / Accessibility | Lighthouse, OWASP ZAP, VoiceOver/TalkBack, Charles Proxy | Производительность, безопасность, a11y |
| L5 | Exploratory / Регрессия | Ручное тестирование, чек-листы | Edge cases, usability, кросс-устройства |
